### STEP 1 — Define the OCREngine interface (Documentation Only)

Here is the conceptual interface definition that will serve as our architectural contract for the remainder of the refactor. This ensures that any engine we plug into the system—whether it’s the existing Tesseract/Paddle or the upcoming Transformers.js engines—will behave predictably.

```typescript
/**
 * OCREngine Interface Reference
 * 
 * The minimal contract required for any OCR engine integration 
 * in the VN-OCR ecosystem. This ensures the capture-to-UI pipeline
 * remains polymorphic and engine-agnostic.
 */
interface OCREngine {
  /** 
   * Unique identifier for the engine. 
   * Matches values in the engine selector (e.g., 'tesseract', 'paddle').
   */
  id: string;

  /** 
   * Human-readable display label for the UI.
   */
  label: string;

  /** 
   * Current initialization state. 
   * Should be true only after load() has successfully completed.
   */
  isLoaded: boolean;

  /** 
   * Asynchronously initializes the engine.
   * Handles model downloads, worker creation, and runtime setup.
   */
  load(): Promise<void>;

  /** 
   * Safely disposes of resources.
   * Terminates workers, nulls sessions, and frees WASM memory.
   */
  dispose(): Promise<void>;

  /** 
   * Primary inference method.
   * @param {HTMLCanvasElement} canvas The image to be processed.
   * @returns {Promise<{ text: string }>} A unified result object.
   */
  recognize(canvas: HTMLCanvasElement): Promise<{ text: string }>;
}
```
